Mobile Bottom Sheet Testing Guide

Test Checklist:

1. Default Position (Collapsed State)

  • ✓ Bottom sheet visible at bottom of screen (80px height)
  • ✓ Only shows NFT title, no description
  • ✓ Single drag handle at top (no duplicates)
  • ✓ No instructional text visible

2. Touch/Drag Responsiveness

  • ✓ Tap anywhere on collapsed sheet → expands to show full details
  • ✓ Drag handle upward → expands sheet smoothly
  • ✓ Drag handle downward when expanded → collapses sheet
  • ✓ Sheet follows finger during drag

3. Auto-Collapse Behavior

  • ✓ Rotate sphere to new item → sheet auto-collapses
  • ✓ New item title appears in collapsed state
  • ✓ Previous expansion state not preserved

4. Visual Design

  • ✓ Dark translucent background (85% opacity)
  • ✓ Backdrop blur effect
  • ✓ Subtle top border (10% white)
  • ✓ Ultra-minimal aesthetic

5. Top Bar Coordination

  • ✓ Top bar visible when sheet collapsed
  • ✓ Top bar hides when sheet expanded
  • ✓ Top bar auto-hides after 3 seconds of inactivity

To Test:

  1. Open http://localhost:3000 on mobile device or browser mobile view
  2. Verify each checklist item above
  3. Report any issues found

Key Implementation Details:

  • Component: /components/mobile/UltraMinimalSheet.tsx
  • Collapsed height: 80px
  • Expanded height: 60-70% of screen
  • Uses react-modal-sheet library
  • Sheet.Backdrop component essential for touch handling
